## BUSINESS RULES (implement as server-side logic, not just UI validation)

1. **Reward Redemption**: An employee can redeem a Reward only if reward.stock &gt; 0 and
   employee.points &gt;= reward.pointsRequired. On redemption: deduct points, decrement stock,
   create a RewardRedemption record, in a single transaction.
2. **Notifications**: Trigger on — new ComplianceIssue created, CSR/Challenge approval
   decision made, policy acknowledgement reminder (e.g. cron/manual trigger for
   unacknowledged policies), badge unlocked. Respect OrgESGConfig / per-user Notification
   Settings for channel (in-app and/or email).
3. **Auto Emission Calculation**: If OrgESGConfig.autoEmissionCalculationEnabled is true,
   creating/linking a Purchase/Manufacturing/Expense/Fleet record auto-generates a
   CarbonTransaction using the matched EmissionFactor (quantity * co2eFactorValue). If
   disabled, allow manual CarbonTransaction entry.
4. **Evidence Requirement**: If OrgESGConfig.evidenceRequirementEnabled is true, an
   EmployeeParticipation record cannot be set to Approved unless proofFileUrl is not null.
   Enforce this in the API layer, not just the form.
5. **Badge Auto-Award**: If OrgESGConfig.badgeAutoAwardEnabled is true, after any action
   that changes an employee&#39;s XP or completed-challenge count, check all Badges&#39; unlockRule
   against the employee&#39;s current stats and auto-create the award + trigger a
   &quot;BadgeUnlocked&quot; notification. If disabled, expose a manual &quot;Award Badge&quot; admin action.
6. **Compliance Issue Ownership**: ownerId and dueDate are required fields at creation
   (enforce with zod + DB constraint). A background/scheduled check (or a computed flag on
   read) marks issues &quot;Overdue&quot; when dueDate &lt; today and status is still Open — feed this
   into Notifications.
7. **Challenge Lifecycle**: Enforce allowed status transitions only:
   Draft -&gt; Active -&gt; UnderReview -&gt; Completed, with Archived reachable from any state.
   Reject invalid transitions at the API layer.
8. **Score Roll-up**: DepartmentScore.totalScore = (environmentalScore * envWeight +
   socialScore * socialWeight + governanceScore * govWeight) / 100, using OrgESGConfig
   weights (default 40/30/30). Overall ESG Score = weighted average of all
   DepartmentScore.totalScore values, weighted by department employeeCount (state this
   assumption clearly in the UI/config).
